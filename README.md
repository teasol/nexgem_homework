# 📑 NEXGEM Algorithm Homework

NEXGEM 기초 알고리즘 과제 제출을 위한 저장소입니다. 리트코드(LeetCode) 문제 풀이를 진행하며, 문제 해결을 위한 사고의 흐름과 최적화 과정을 기록합니다.

## 📂 Folder Structure

과제는 주차별 폴더로 관리하며, 각 문제별로 하위 폴더를 생성하여 관리합니다.

```text
/
├── week1/                 # 1주차 과제 (예: Array, String)
│   ├── Problem_Name_1/    # 문제별 폴더
│   │   ├── debug.py       # 디버깅용 파일
│   │   ├── solution.py    # 최종 제출 코드
│   │   └── README.md      # 시도한 기록, 분석 및 리뷰
│   └── ...
├── week2/                 # 2주차 과제 (예: Stack, Queue)
└── README.md              # 메인 가이드 (본 파일)

```

## 📝 Submission Guidelines

### 1. Code Versions

* 단순히 정답을 맞히는 것이 목적이 아니라, **최적화 과정**을 보여주는 것이 중요합니다.
* **Trial 1:** 가장 먼저 떠오른 직관적인 해결법을 작성합니다.
* **Trial 2~N-1:** 다른 방법으로 시도해보면서 각 시도마다 의도한 방법 및 개선된 점을 적습니다.
* **Trial N:** Accepted 알고리즘을 분석하고 어떤 점을 배웠는지 리뷰합니다.

### 2. Problem Review (README.md in problem folder)

각 문제 폴더의 README에는 다음 내용을 포함합니다.

* LeetCode 문제 링크
* 각 버전의 Trial History
* 최종 알고리즘의 시간 복잡도 설명

매번 시간복잡도를 분석할 필요는 없으나, 최종 알고리즘의 시간 복잡도 설명은 필수입니다.
단순 문법 오류는 기록할 필요 없습니다.
기본적인 디버깅 후에 문법 오류가 없음을 확인하고 commit하는 습관을 들이도록 합시다.

## 🚀 Requirements

* 기초 알고리즘 원리에 대한 이해
* Python 3.7 이상
* 비판적 사고를 통한 코드 최적화
