# 📑 NEXGEM Algorithm Homework

NEXGEM 기초 알고리즘 과제 제출을 위한 저장소입니다. 리트코드(LeetCode) 문제 풀이를 진행하며, 문제 해결을 위한 사고의 흐름과 최적화 과정을 기록합니다.

## 🚀 How to Start (과제 시작 방법)

본 저장소는 **Template Repository**로 설정되어 있습니다. 아래 절차에 따라 개인 저장소를 생성하고 과제를 진행하세요.

1. **Template 사용:** 상단의 **[Use this template]** → **[Create a new repository]** 버튼을 클릭합니다.
2. **저장소 설정:**  
    * **Repository name:** 자유롭게 설정  
    * **Visibility:** 반드시 **Private**으로 설정합니다. (다른 참여자와 코드가 공유되지 않도록 하기 위함입니다.)  
3. **Collaborator 초대:** 과제 검토를 위해 생성한 개인 저장소의 **Settings** → **Access/Collaborators** 메뉴에서 관리자(`teasol`)를 초대해 주세요.

## 📂 Folder Structure

과제는 주차별 폴더로 관리하며, 각 문제별로 하위 폴더를 생성하여 관리합니다.

```text
/
├── week1/                 # 1주차 과제 (예: Array)
│   ├── Problem_Name_1/    # 문제별 폴더
│   │   ├── debug.py       # 디버깅용 파일 (선택)
│   │   ├── solution.py    # 최종 제출 코드
│   │   └── README.md      # 시도한 기록, 분석 및 리뷰
│   └── ...
├── week2/                 # 2주차 과제 (예: Linked List)
└── README.md              # 메인 가이드 (본 파일)

```

## 📝 Submission Guidelines

각 문제 폴더의 `README.md`에는 다음 내용을 포함해야 합니다.

* **LeetCode 문제 (링크)**
* **Trial History:** 시도한 모든 기록을 지우지 말고 이어서 작성합니다.
* **Approach:** 각 버전(Commit)마다 어떤 방식으로 접근했는지 설명합니다.
* **Complexity:** 최종 알고리즘의 **시간 복잡도(Time Complexity)** 설명은 필수입니다. **공간 복잡도(Space Complexity)**는 중요한 경우만 작성.
* **Review:** 문제 분석 및 최종 리뷰를 작성합니다.

> [!IMPORTANT]
> 단순 문법 오류는 기록할 필요 없습니다. 기본적인 디버깅을 마친 후 논리적 흐름이 변할 때마다 기록하며, 단순히 정답을 맞히는 것이 아니라 **최적화 과정**을 보여주는 것이 중요합니다.

## 🤖 Generative AI Usage Guidelines

본 과제의 목적은 **'문제를 해결하는 사고 과정'**을 훈련하는 것입니다. AI를 도구로 활용하되, 아래 원칙을 준수해야 합니다.

### 1. 허용되는 범위 (Recommended)

* **개념 설명 및 원리 학습:** 특정 알고리즘의 작동 원리를 이해하기 위해 설명을 요구하는 경우.
* **코드 디버깅 힌트:** 논리적 오류를 찾지 못할 때 에러 메시지를 바탕으로 힌트를 얻는 경우.
* **주석 및 README 작성:** 코드 설명 및 주석 작성에는 AI 활용 적극 권장.
* **복잡도 검증:** 본인이 계산한 시간/공간 복잡도가 맞는지 확인하는 경우.

### 2. 제한되는 범위 (Restricted)

* **정답 코드 복사 및 붙여넣기:** AI가 생성한 전체 코드를 무비판적으로 제출하는 행위는 금지합니다.
* **검증 없는 수용:** AI의 답변은 틀릴 수 있습니다. 반드시 본인이 직접 검증하고 이해한 내용만 기록해야 합니다.

### 3. AI 활용 시 기록 방법 (Disclosure)

AI의 도움을 받았다면 `README.md` 내 **Approach**에 반드시 명시해야 합니다.

* *작성 예시: "1시간 동안 해결하지 못해 Gemini에게 힌트를 요청함. Two-pointer 기법을 새로 알게 되어 적용해 봄."*

### 4. 핵심 원칙: "Think First, Ask Later"

* 최소 **20분 이상**은 외부 도움 없이 스스로 분석하고 설계를 시도합니다.
* 질문 시에는 "정답을 알려줘"가 아니라, **"내 접근 방식에서 시간 복잡도를 더 줄일 수 있는 아이디어가 있을까?"**와 같이 구체적으로 질문합니다.
* AI가 작성한 로직은 반드시 한 줄씩 직접 타이핑하며 분석해야 합니다.

* Python 3.7 이상
* 비판적 사고를 통한 코드 최적화 및 복잡도 분석 능력
