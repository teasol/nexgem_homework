# 📑 NEXGEM Algorithm Homework

NEXGEM 기초 알고리즘 과제 제출을 위한 저장소입니다. 리트코드(LeetCode) 문제 풀이를 진행하며, 문제 해결을 위한 사고의 흐름과 최적화 과정을 기록합니다.

## 📂 Folder Structure

과제는 주차별 폴더로 관리하며, 각 문제별로 하위 폴더를 생성하여 관리합니다.

```text
/
├── Week1/                 # 1주차 과제 (예: Array, String)
│   ├── Problem_Name_1/    # 문제별 폴더
│   │   ├── attempt_1.py   # 첫 번째 시도 (Brute-force 등)
│   │   ├── attempt_2.py   # 성능 개선 버전
│   │   ├── final.py       # 최종 제출 코드
│   │   └── README.md      # 문제 분석 및 회고
│   └── ...
├── Week2/                 # 2주차 과제 (예: Stack, Queue)
└── README.md              # 메인 가이드 (본 파일)

```

## 📝 Submission Guidelines

### 1. Code Versions

* 단순히 정답을 맞히는 것이 목적이 아니라, **최적화 과정**을 보여주는 것이 중요합니다.
* **Attempt 1:** 가장 먼저 떠오른 직관적인 해결법을 작성합니다.
* **Attempt 2~N:** 시간/공간 복잡도를 개선하거나 가독성을 높인 코드를 작성합니다.
* **Final:** 가장 효율적이라고 판단되는 최종 코드를 제출합니다.

### 2. Commenting Rules

* **[Mandatory] All comments in the code must be written in English.**
* 코드의 핵심 로직, 복잡도 분석(Time/Space Complexity), 그리고 특정 라이브러리나 알고리즘을 사용한 이유를 영문 주석으로 상세히 설명하세요.

### 3. Problem Review (README.md in problem folder)

각 문제 폴더의 README에는 다음 내용을 포함합니다.

* LeetCode 문제 링크
* 해당 알고리즘의 시간/공간 복잡도 설명
* 이전 버전에서 무엇을 개선했는지에 대한 회고

## 🚀 Requirements

* 기초 알고리즘 원리에 대한 이해
* Python/Java/C++ 등 선택한 언어의 컨벤션 준수
* 비판적 사고를 통한 코드 최적화
